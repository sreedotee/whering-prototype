Whering · Visual Language v1

The system.

A tinted-surface design system for Whering. Each section of the app sits on one soft hue wash. Type is always black. Components are hue-agnostic — drop them anywhere, they retint themselves.

Principles Hues Type Radii Spacing Icons Components Hue in use Rules for agent Migration checklist

01 · Principles

Five things that are always true.

One hue per section. Discover / Search / Try On / Notifications live on cyan. Profile lives on lime. Stats moments pop to violet. Never mix two hues in one screen.
Type is black. No colored text, ever. Hierarchy comes from size and weight, not color.
Surfaces are quiet. Cards are a half-step lighter than the wash, not pure white. White is reserved for modals and input fields.
Accent is an event. The saturated hue (cyan #1DB5FF, lime #9BC926 etc.) appears only as a filled tag, a tiny indicator, or a chart bar. Never as a button or a background.
Radii are one scale. 8 / 14 / 20 / 28 / pill. Nothing else. Ever.

02 · Hues

Five washes. Assigned by section.

Each hue has four roles. bg is the full-screen wash. surface is cards and rows on top of it. edge is hairlines. accent is the one true saturated tone — used sparingly as a filled tag or indicator, never as a button or surface.

Gap between cards in a grid: 12px (s-3). Gap between chips/buttons: 8px (s-2).

06 · Icons

One icon family. 24px. Stroke 2. Round caps.

Every icon uses the same stroke weight and cap style. Inactive state is --ink-3, active is --ink-0. No filled icons. No two-tone. No color. Icons never carry information that isn't also in a label or aria.

Use Lucide as source. It already matches these rules.
Strip all existing emoji / Phosphor / Heroicons usage. They mix stroke weights and break the line.

09 · Rules for the agent

Implementation contract.

Hard constraints. An implementation that breaks any of these is failing the system.

Import tokens.css + components.css first. Before any other stylesheet. They define the variables every component reads.
Apply hue at the screen root. Every top-level screen component gets className="hue-cyan" (or lime, violet, orange, oat). Not inline styles. Not per-element.
Read from variables. Any new component must use var(--surface), var(--bg), var(--edge), var(--accent), var(--ink-0) etc. Never hardcode hex.
Type sizes only from scale. font-size must be one of 12, 14, 16, 20, 24, 32. Nothing else.
Radii only from scale. 8, 14, 20, 28, 9999. Nothing else.
Spacing from the 4pt grid. 4, 8, 12, 16, 20, 24, 32, 40, 48. No 5, 10, 14, 18, 22 etc.
No colored text. Text is --ink-0, --ink-1, --ink-2, --ink-3. Cyan/lime/violet never touch text.
No colored buttons. Primary button is always black. Surface button is --surface. Never fill a button with cyan/lime/violet.
No gradient surfaces. Flat fills only. Gradients are reserved for the subtle image-bottom shadow on garment thumbs.
No emoji in UI. Use Lucide icons. Emoji only ever appears in user-generated content.
No pure-white backgrounds on hue screens. Surfaces are --surface (a half-tint of the wash). White is reserved for modal sheets and input focus states.
No drop shadows on cards. Cards sit on a tinted wash — that's the depth. Shadows are only on modals (--e3) and floating action buttons (--e2).

10 · Migration checklist

Things to rip out of the current codebase.

Specific issues the agent needs to fix, in priority order. Ship these in this sequence — do not start component work before tokens land.

  1. Install tokens. Replace the existing :root in index.css with tokens.css. Audit all references to CSS variables — rename anything that conflicts.
  2. Strip old radii. Grep border-radius across the project. Any value not in {8, 14, 20, 28, 9999} gets remapped. Common offenders: 4, 6, 10, 12, 16, 24.
  3. Strip old type sizes. Grep font-size. Any value not in {12, 14, 16, 20, 24, 32} gets remapped. 11 → 12. 13 → 14. 15 → 16. 18 → 16 or 20 (pick by context). 22 → 20 or 24. 28 → 24 or 32.
  4. Strip all gradients except the image-bottom shadow inside .card--image.
  5. Strip all colored text. Grep color:. Replace any non-ink color with an ink token. Body #333-ish → --ink-1. Muted → --ink-2. Faint → --ink-3.
  6. Strip all colored fills from buttons. Grep .btn, button, [class*="button"]. Fills must be --ink-0 (primary), --surface (secondary), or transparent (ghost).
  7. Wrap each screen in a hue scope. Add className="hue-cyan" to Discover / Search / Try On / Notifications / Add-flow root. hue-lime to Profile root and any friend/follower screen. hue-violet to the Stats section inside Profile. hue-oat to Settings.
  8. Replace emoji with Lucide. npm i lucide-react. Grep the codebase for emoji characters and swap each to a Lucide icon at 24px stroke 2.
  9. Remove card shadows. Grep box-shadow. Remove from everything except .sheet, .modal, and any floating action button.
  10. Tab bar. Single pattern — 5 icons, active state is black-filled pill at 14px radius. No labels. No badges. No color.
  11. Modals. All modals use .sheet — white paper, 28px top radius, grab handle, 24/700 centered title. Rows inside modals sit on the parent hue wash, not on white.
  12. Empty states. Switch to hue-oat scope. One editorial line at 32/700. One action button. No illustrations (for now).
